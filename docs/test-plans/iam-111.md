# Test Plan: IAM-111 Ticket Card Redesign

## Scope

Verify the user-facing ticket card redesign on the live Kanban Board app at `http://127.0.0.1:5174`.

In scope:

- Board renders with visible ticket cards.
- Cards show a colored left border accent tied to ticket type.
- Hovering a card reveals a drag handle in the header area.
- Footer uses a compact single-line layout with a priority dot, tags, and metadata.
- Cards still open ticket detail and still support drag-and-drop between columns.
- Board palette remains consistent with the established warm cream surface.

Out of scope:

- Source-code or DOM-implementation inspection.
- Backend persistence details beyond what is visible in the running UI.
- Mobile layout behavior.

## Test Approach

- Black-box Playwright execution against `http://127.0.0.1:5174`.
- Visual inspection with screenshots and accessibility snapshots.
- No source-code inspection.

## Test Data Strategy

- Use existing board data only.
- Do not create or delete tickets.
- If drag-and-drop changes board state, return the moved card to its original column when feasible.

## Execution Results

Run date: 2026-04-18
Execution mode: Playwright browser validation against `http://127.0.0.1:5174`, with fallback execution on `http://localhost:5173` after the requested port refused connections
Overall result: Mixed

### Observed Results

- TC-01: FAIL. Navigating to `http://127.0.0.1:5174` returned `ERR_CONNECTION_REFUSED`, so the board did not load at the requested address.
- TC-02: PASS. Visible cards showed distinct left-edge accents rather than one shared color: the chore card in Backlog used a muted gray accent, bug cards used warm orange-red accents, and the task card in Done used a blue accent.
- TC-03: PASS. On `SBFF-18`, the drag-handle icon was not visible before hover and appeared in the top-right corner after hover.
- TC-04: PASS. Card footers used a small colored priority dot rather than a text badge.
- TC-05: PASS. Footer metadata remained compact on one row in sampled cards, with tags and counts aligned on the same line instead of stacking into separate rows.
- TC-06: PASS. `SBFF-22` showed exactly two visible tags (`whitebox-utc`, `coverage`) followed by `+2` for overflow.
- TC-07: PASS. Clicking `SBFF-18` opened the ticket detail modal and updated the URL to `?ticket=SBFF-18`.
- TC-08: PASS. A real pointer drag moved `SBFF-18` from `To Do` to `In Progress`; the card was then moved back to `To Do` to restore the board state.
- TC-09: PASS. The board retained its warm cream page background and the column header colors remained consistent with the existing palette.
- Additional observation: no browser console warnings or errors were emitted during the fallback execution on `localhost:5173`.

### Evidence

- Baseline full-board screenshot captured the warm cream canvas, unchanged column treatments, and visible card accents.
- Before/after hover screenshots of `SBFF-18` confirmed the drag handle appears only on hover.
- A card screenshot of `SBFF-22` confirmed the compact footer, small priority dot, two visible tags, `+2` overflow indicator, and `2/2` counter on one line.
- Accessibility snapshot of the modal confirmed `SBFF-18` opens in a dialog after clicking the card.
